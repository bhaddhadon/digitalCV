Driver Onboarding Auto-Approval

Problem Statement

Driver Onboarding process's turnaround time is a bit long, especially in a big market with huge population. The bottleneck is a manual reviewing process not being scalable and, more often than not, resulting in the application being disapproved due to some flaws in the uploaded documents.

Solution

An automated process mimicking checks performed by humans. The process combines the use of

  • 1. OCR of info on documents/images and make sure the info is matched/ mostly matched with the info submitted from the registration portal.
  • 2. Object Detections of items in both positive and negative lists to make sure the image meets the requirements.
  • 3. Image Quality measures: blurriness/ overexposure/ underexposure
  • 4. Identity Matching: between submitted images and ID documents
  • 5. Document Spoof Detection: PDF metadata/ Image splicing/ Copy-move-forgery

Key Model Measurements:

  • OCR Match Rate: 80%+ Recall given minimum acceptable Levenshtein Match Rate
  • Custom Object Detection: 95% mAP
  • Generic Object Detection: 90%+ Recall

Key Business Objective:

  • Auto-verified Rate
  • Turnaround time reduction
Benefits
  • Lower turnaround time
  • Higher conversion rate
  • Better customer experience - adviser with relevant product knowledge and accustomed to advising customer with similar need stage
Evaluation
  • Auto-verified registration apps done within a day
  • The auto-verification process can be halted when there're a new essential requirements added
  • Auto-verified rate is at 20%. The number can go higher with rules simplification/relaxation.
Next Steps
  • Rules simplification/relaxation
  • Move the quality assurance process to frontend, to minimize disapproval
